스프링부트 파일 업로드

업로드 수행은 부트에 내장된 Tomcat 의 API 를 그대로 이용합니다.

객체이름은 MultiPart 실제 업로드 되는 파일의 정보를 관리하는 객체

업로드 흐름을 간단히 설명, 파일업로드 폼 요청(컨트롤러에서 view 준비) -->
업로드 요청(Ajax 로 처리,, 일반폼인 경우 encType 설정 반드시 해야함) --> 컨트롤러에서 업로드 요청 처리
매핑 및 업로드 처리(MultiPart 객체 이용) -->  클라이언트에게 결과 전송

부트에서는 위 업로드를 처리하기 위한 사전 준비가 필요함

준비될 내용으로는, MultiPart 사용 여부 설정(true), 업로드될 파일의 저장될 Root 경로(절대 경로), 한 번에 최대 업로드 가능 용량 설정
한 파일에 대한 최대 크기 설정등임,, 추가 설정은 구글링,,

이 설정은 모두 .properties 파일에 설정함

이렇게 설정한 후체 업로드 요청이 왔을 때, 어노테이션을 이용해서 컨트롤러는 Root 경로를 얻어내고, 필요에 따라서 서브 folder 등을 생성해서
해당 folder 에 파일을 저장함

옵션사항으로는,
1. 서브폴더를 생성할지의 여부(만약 생성하지 않으면, 하나의 폴더에는 Max 로 저장될 갯수 및 용량이 지정되있으므로,
이를 넘길시 예외 or 저장되지 않았음의 오류 발생함)

2. 파일명변조 여부 및 업로드되는 파일의 종류(확장자 기준) 지정,,

3. 기존 파일이 존재 하는 경우에, 같은 파일이 업로드시 어떻게 할 지의 여부(덮어씀 or 파일명틀리게 해서 재저장등,, 여기선 재저장)

위의 옵션들을 고려해서 파일업다운로드를 처리할 예정
